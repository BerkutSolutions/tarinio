# Runbook Anti-DDoS

Эта страница относится к текущей ветке документации.

Короткий порядок действий при всплеске:

1. Проверить `Dashboard` и `Events`.
2. Оценить источник и тип нагрузки.
3. Включить или усилить защитные режимы.
4. Проверить влияние на нормальный трафик.
5. Зафиксировать итог в журнале изменений.

# Русская wiki

Эта страница относится к текущей ветке документации.

Русская ветка документации TARINIO предназначена для повседневной работы с продуктом: от первого входа до эксплуатации, расследования инцидентов, обновлений и восстановления после сбоев.

## Для кого эта документация

- Для операторов, которые работают в интерфейсе и выпускают ревизии.
- Для инженеров внедрения, которые готовят стенды и продуктивные среды.
- Для администраторов, которые отвечают за безопасность, резервное копирование и сопровождение.

## Что входит в русскую wiki

- описание платформы и её архитектуры;
- обзор интерфейса и рабочих сценариев;
- каталог API;
- инструкции по развёртыванию, обновлению и откату;
- эксплуатационные и операторские руководства;
- документы по безопасности, высокой доступности и наблюдаемости.

## С чего начать

1. Прочитайте [Архитектуру](architecture-docs/architecture.md), чтобы понять границы продукта.
2. Откройте [Интерфейс](core-docs/ui.md), чтобы увидеть реальные разделы и сценарии работы.
3. Для внедрения перейдите в [Развёртывание](core-docs/deploy.md), затем в [Безопасность](core-docs/security.md).
4. Для ежедневной эксплуатации используйте [Эксплуатацию](core-docs/runbook.md), [API](core-docs/api.md) и операторские документы.

## Изменения в релизах `3.0.3`-`3.0.5`

### `3.0.3`
- Исправлены сценарии первой установки и первого запуска: порядок компиляции и применения, выпуск `ACME http-01`, переход на `HTTPS` и стабильность onboarding-потока.
- Импорт `.env` и `.json` переведён в staged-модель с одной финальной компиляцией и применением без каскада промежуточных ревизий.
- Улучшена диагностика рантайма: ошибки `waf-runtime-l4-guard` и неуспешного применения показывают фактическую причину.

### `3.0.4`
- Доведены сценарии установки и обновления: безопасное сохранение существующих секретов и корректное завершение установки без ложного `[FAIL]`.
- Механизм anti-bot получил промежуточную страницу проверки браузера с маршрутом подтверждения и возвратом пользователя на исходный URL.
- Зафиксированы правила обхода для административного хоста, чтобы служебные маршруты не блокировались anti-bot и rate-limit логикой.

### `3.0.5`
- Release-синхронизация версии, UI и документации доведена до единого состояния `3.0.5`.
- Страница проверки переведена на полноценный поток перенаправления браузера с визуальным статусом проверки.
- Служебный admin-трафик на публичном host исключён из пользовательской request/security аналитики и метрик панели мониторинга.

# Руководство по усилению защиты

Эта страница относится к текущей ветке документации.

## Цель

Этот документ задаёт рекомендуемый базовый уровень усиления защиты для промышленного контура поверх базовых инструкций по развёртыванию.

## Административная граница

Рекомендуемый базовый уровень:

- ограничивать доступ к control-plane доверенными сетями;
- публиковать административные конечные точки только по HTTPS;
- не выставлять административный интерфейс напрямую в интернет без контроля;
- использовать именованные операторские аккаунты и второй фактор.

## Секреты

Рекомендуемый базовый уровень:

- не хранить секреты в публичном репозитории;
- менять стартовые секреты до использования в промышленной среде;
- считать экспорт сертификатов и API-токены чувствительными артефактами;
- использовать защищённое хранилище секретов при возможности.

## Сетевая сегментация

Рекомендуемый базовый уровень:

- отделять публичный входящий трафик от административного доступа;
- держать PostgreSQL и Redis только во внутренних сетях;
- использовать правила межсетевого экрана хоста вместе с защитой TARINIO;
- публиковать только действительно нужные порты.

## Безопасность изменений

Рекомендуемый базовый уровень:

- все значимые изменения политик проводить через компиляцию и применение ревизий;
- держать ревизии готовыми к откату;
- использовать пакетную запись для крупных изменений инициализации;
- проверять систему после каждого изменения, влияющего на безопасность.

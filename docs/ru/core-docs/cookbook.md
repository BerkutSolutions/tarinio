# Практические сценарии эксплуатации

## Назначение

Этот сборник практик собирает типовые операторские задачи в короткие повторяемые последовательности.

## Добавить новый сайт

1. Создать сайт.
2. Создать upstream-сервис.
3. Привязать TLS при необходимости.
4. Скомпилировать ревизию.
5. Применить ревизию.
6. Проверить `/healthcheck` и `Requests`.

## Безопасно выкатить большой пакет изменений

1. Сохранить изменения с `X-WAF-Auto-Apply-Disabled: true`.
2. Проверить ожидаемое состояние.
3. Скомпилировать один раз.
4. Применить один раз.
5. Проверить `Dashboard`, `Events` и метрики.

## Аккуратно ужесточить CRS

1. Зафиксировать текущую ревизию.
2. Изменить политику, связанную с CRS.
3. Выполнить компиляцию и применение.
4. Следить за ложными срабатываниями в `Requests` и `Events`.
5. Добавлять исключения только там, где это действительно нужно.

## Разобрать заблокированный запрос

1. Найти запрос в `Requests`.
2. Связать его с `Events`.
3. Понять, кто блокировал: WAF, ограничение скорости или Anti-DDoS.
4. Решить, нужна ли донастройка политики или блокировка корректна.

## Включить инспекцию WebSocket

1. Открыть профиль сайта → вкладка `WebSocket`.
2. Включить `use_websocket_inspection`.
3. Задать ограничения фреймов (`max_frame_size`, `max_message_size`) под ожидаемый трафик.
4. При необходимости добавить запрещённые паттерны.
5. Скомпилировать и применить ревизию.
6. Наблюдать за `Events` на предмет срабатываний WebSocket.

## Включить mTLS для сайта

Входящий mTLS (клиент → WAF):

1. Открыть профиль сайта → вкладка `mTLS`.
2. Включить `use_mtls`.
3. Указать `mtls_vault_path` — путь к ClientCA в Vault.
4. При необходимости включить `mtls_require_client_cert`.
5. Скомпилировать и применить ревизию.
6. Проверить, что соединения без сертификата отклоняются.

Исходящий mTLS (WAF → upstream):

1. Включить `use_upstream_mtls`.
2. Указать `upstream_mtls_cert_vault_path` и `upstream_mtls_key_vault_path`.
3. Скомпилировать и применить ревизию.
4. Убедиться, что upstream принимает клиентский сертификат WAF.

## Включить Virtual Patching

1. Открыть профиль сайта → раздел `Virtual Patches`.
2. Добавить правило SecRule в поле `rule` и описание в `description`.
3. Убедиться, что правило синтаксически корректно (ModSecurity SecRule).
4. Скомпилировать и применить ревизию.
5. Проверить в `Events`, что правило срабатывает на нужные запросы.

## Настроить Geo Time Windows

1. Открыть профиль сайта → вкладка `Geo Time Windows`.
2. Добавить запись: страна (`country_code`), временной диапазон, действие (`allow`/`deny`).
3. Скомпилировать и применить ревизию.
4. Проверить в `Requests`, что трафик из указанных регионов обрабатывается согласно политике.

## Настроить обнаружение credential stuffing

1. Убедиться, что на сайте задан `auth_endpoint_path`.
2. Настроить порог `auth_failure_threshold` — количество 401-ответов, после которого IP блокируется.
3. Скомпилировать и применить ревизию.
4. Проверить сигналы `auth_failures` в Sentinel и `Events`.

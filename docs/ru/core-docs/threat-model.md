# Модель угроз

Эта страница относится к текущей ветке документации.

Релизный baseline для этой ревизии документа: `3.0.6`.

## Назначение

Этот документ объясняет, что именно TARINIO помогает защищать, где проходят доверительные границы и какая ответственность остаётся у оператора.

## Защищаемые активы

TARINIO в первую очередь помогает защищать:

- входящий контур приложений;
- обработку запросов под управлением политик;
- контролируемое внедрение настроек защиты;
- операторскую видимость событий безопасности и внедрения изменений.

В текущем релизе к защищаемым операционным активам также относятся:

- целостность ревизий и безопасность rollout-потока;
- целостность состояния control-plane в `PostgreSQL`;
- целостность наблюдаемости запросов/событий в `OpenSearch` и опционально в `ClickHouse`;
- целостность координации высокой доступности в `Redis`.

## Основные границы доверия

- публичный клиент -> входящий контур рантайма;
- рантайм -> upstream-приложения;
- оператор -> control-plane;
- control-plane -> `PostgreSQL` / `Redis` / API рантайма;
- рантайм и control-plane -> backend запросов `OpenSearch` / `ClickHouse` (если включены).

## Какие угрозы TARINIO помогает закрывать

- типовой веб-атакующий трафик через WAF/CRS;
- вредоносные шаблоны запросов и сценарии, похожие на brute force;
- враждебные L4/L7-шаблоны трафика через Anti-DDoS;
- опасный дрейф конфигурации через поток изменений на основе ревизий.

В текущем релизе TARINIO также снижает риски:

- split-brain для compile/apply в режиме высокой доступности за счёт распределённых блокировок;
- потери целостности в контролируемых потоках миграции/синхронизации истории запросов.

## Разделение ответственности

TARINIO не заменяет:

- безопасный код приложений;
- сетевую сегментацию;
- управление секретами;
- backup и disaster recovery;
- усиление защиты хоста и контейнеров.

Зона ответственности оператора дополнительно включает:

- управление lifecycle и retention для `OpenSearch` и `ClickHouse`;
- регулярные backup/restore-проверки всех включённых backend (`PostgreSQL`, `Redis`, `OpenSearch`, `ClickHouse`);
- валидацию результатов синхронизации данных после апгрейдов и инцидентов.

## Нецели

TARINIO сам по себе не является:

- SIEM;
- полноценной IAM/directory-service системой;
- EDR-продуктом;
- заменой практик secure software development;
- универсальным оркестратором глобальной синхронизации для произвольных внешних хранилищ.

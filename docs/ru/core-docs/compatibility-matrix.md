# Матрица совместимости

Эта страница относится к текущей ветке документации.

## Назначение

Матрица фиксирует окружения и сочетания компонентов, вокруг которых TARINIO `3.0.2` проектируется, проверяется и поддерживается по SLA.

## Платформа выполнения

Поддерживаемый базовый сценарий:

- Linux-хосты для промышленного развёртывания;
- Docker Engine с плагином Docker Compose;
- контейнерные образы на базе Debian для `control-plane` и встроенного PostgreSQL;
- рантайм на базе NGINX и ModSecurity.

## Сервисы данных

Поддерживаемый базовый набор:

- PostgreSQL `15`
- Redis `7`

Практическое правило:

- держаться близко к документированным мажорным версиям, если новые версии не были отдельно проверены в стендовой среде.

## Режимы развёртывания

Документированные и поддерживаемые сценарии:

- однокомпонентное развёртывание через compose;
- стандартное развёртывание с хранилищем на PostgreSQL;
- кластер `control-plane` в режиме высокой доступности с общими PostgreSQL и Redis;
- стенд высокой доступности с профилем наблюдаемости;
- обновления через сценарий AIO.

## Поддерживаемые каналы версий

- `Current`: полный функциональный и security support.
- `Stable`: bugfix + security support без расширения функциональности.
- `LTS 2.0`: security и критичная устойчивость до 30 апреля 2027 года.

## Браузеры

Интерфейс рассчитан на актуальные браузеры с регулярными обновлениями:

- актуальные Chrome и Chromium;
- текущий Microsoft Edge;
- текущий Firefox.

## Что не покрывается матрицей

- произвольные Kubernetes-дистрибутивы без отдельной валидации;
- неподдерживаемые мажорные версии БД;
- ручная модификация runtime-образов и compose-профилей вне документированных путей.

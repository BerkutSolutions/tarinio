# Карта соответствия требованиям

## Назначение

Эта страница помогает командам безопасности и комплаенса понять, как возможности TARINIO v1.3.5 соотносятся с типовыми классами требований.

## Важное уточнение

Это функциональная карта соответствия, а не заявление о формальной сертификации.

## OWASP-ориентированное соответствие

TARINIO помогает в областях:

- проверка запросов и применение политик (WAF/CRS, ModSecurity);
- видимость трафика и корреляция событий;
- безопасное внедрение изменений безопасности через revision workflow;
- ограничение частоты запросов и подавление враждебного трафика;
- обнаружение подбора учётных данных (credential stuffing, OWASP ASVS 2.2);
- блокировка по TLS fingerprint JA3/JA4 (снижение рисков автоматизированных атак);
- виртуальные патчи (virtual patches) для временного закрытия известных уязвимостей;
- инспекция WebSocket-трафика;
- защита от HTTP Request Smuggling через strict parsing;
- взаимная аутентификация mTLS (входящая и исходящая).

## Операционное соответствие требованиям безопасности

TARINIO помогает поддерживать контрольные меры вокруг:

- администрирование по принципу минимальных привилегий;
- аудитируемость изменений (revision log, apply history);
- планирование резервного копирования и восстановления;
- защищённый административный доступ;
- наблюдаемые операции безопасности;
- управление секретами через Vault (mTLS certs, logging credentials);
- геозависимые политики доступа с временными окнами (Geo Time Windows).

## Governance Value

Документация и рабочие процессы TARINIO особенно полезны, когда команды должны предоставить:

- свидетельства контролируемого управления изменениями;
- свидетельства готовности к откату;
- свидетельства мониторируемой входящей защиты;
- свидетельства операторских runbook и процедур восстановления.

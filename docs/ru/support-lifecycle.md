# Политика поддержки и жизненного цикла

Эта страница относится к текущей ветке документации.

TARINIO следует модели активного развития: основной поддерживаемой версией считается текущий релиз, а обновление является нормальной частью эксплуатации, а не исключением.

## Подход к релизам

- новый релиз становится основной поддерживаемой версией;
- обновление предполагается через `scripts/install-aio.sh` или эквивалентный документированный сценарий;
- безопасность обновления обеспечивается резервным копированием, миграциями и проверками после установки;
- длительная параллельная поддержка старых релизных веток не является базовым сценарием.

## Что это означает для операторов

- планируйте работу на актуальной валидированной версии;
- не рассчитывайте на долгую эксплуатацию устаревших сборок без обновлений;
- включайте обновление и откат в стандартный эксплуатационный процесс.

## Поддерживаемый сценарий эксплуатации

Поддерживаемым считается сценарий, в котором:

- используется документированный Docker- или Docker Compose-профиль;
- обновление выполняется штатным способом;
- перед обновлением создаётся резервная копия;
- runtime не редактируется вручную в обход продуктовых процессов.

## Почему обновление считается безопасным

В версии `2.0.5` это обеспечивается следующими механизмами:

- AIO-установщик создаёт предварительные резервные копии;
- миграции PostgreSQL версионированы;
- перенос legacy-состояния выполняется без разрушения данных;
- можно включить строгую проверку после обновления;
- обновление HA-кластера можно проверять поэтапно, без потери доступности API.

## Ответственность оператора

Оператор должен:

- держать артефакты развёртывания в соответствии с текущим релизом;
- выполнять резервное копирование и контрольные проверки;
- заранее оценивать запас ресурсов перед обновлением;
- иметь проверенную точку отката.

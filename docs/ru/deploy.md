# Развёртывание

Эта страница относится к текущей ветке документации.

Документ описывает практические варианты развёртывания TARINIO для локальных стендов, тестовых сред и продуктивных инсталляций.

## Что разворачивается

Минимальный стек включает:

- `control-plane`;
- `runtime`;
- `postgres`;
- сети, тома и конфигурацию выбранного профиля.

В зависимости от сценария могут добавляться `redis`, балансировщик API, инструменты наблюдаемости и вспомогательные сервисы.

## Варианты запуска

### AIO-установка одной командой

```bash
curl -fsSL https://raw.githubusercontent.com/BerkutSolutions/tarinio/main/scripts/install-aio.sh | sh
```

Этот вариант подходит для:

- быстрого знакомства с продуктом;
- демонстрационных стендов;
- тестовых сред с минимальной подготовкой.

### Docker Compose

Основные профили развёртывания описаны в документах:

- `deploy/compose/README.md`
- `deploy/compose/default/README.md`
- `deploy/compose/auto-start/README.md`

Пример базового запуска:

```bash
docker compose -f deploy/compose/default/docker-compose.yml up -d --build
```

## Что проверить после запуска

После первого старта обычно проверяют:

- страницу входа `/login`;
- системный маршрут `/healthz`;
- доступность HTTP и HTTPS на портах runtime;
- корректность перенаправления в первичную настройку, если система ещё не инициализирована.

## Подготовка к продуктивной среде

До ввода в эксплуатацию рекомендуется подготовить:

- DNS-записи и сетевые правила;
- отдельные секреты для среды;
- стратегию TLS: импорт, self-signed, ACME `http-01` или `dns-01`;
- политику хранения логов, событий и аудита;
- схему доступа операторов;
- сценарий резервного копирования и восстановления;
- порядок применения изменений и отката.

## Рекомендации для production

- Не используйте значения по умолчанию для секретов.
- Ограничьте сетевой доступ к control-plane.
- Используйте HTTPS для административного контура.
- Явно настройте trusted proxies.
- Подготовьте резервное копирование томов и секретов.
- Проверьте, что сроки хранения данных и проверка обновлений соответствуют вашей политике эксплуатации.
- Заранее определите порядок отката и ответственных за него.

## Первичная настройка

1. Откройте `/login`.
2. Если система не инициализирована, пройдите первичную настройку.
3. Создайте первого администратора.
4. Создайте первый сайт и upstream.
5. Настройте TLS.
6. Убедитесь, что проверка состояния проходит успешно.
7. Выпустите и примените первую ревизию.

## Как понять, что развёртывание прошло успешно

Установку можно считать успешной, если:

- `/healthz` отвечает стабильно;
- вход и первичная настройка работают без ошибок;
- `GET /api/app/meta` возвращает версию `2.0.8`;
- удаётся создать сайт и выпустить рабочую ревизию;
- после применения ревизии runtime обслуживает нужный хост;
- в интерфейсе появляются события, запросы и аудит.

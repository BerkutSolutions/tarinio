# Практические сценарии эксплуатации

Эта страница относится к текущей ветке документации.

## Назначение

Этот cookbook собирает типовые операторские задачи в короткие повторяемые последовательности.

## Добавить новый сайт

1. Создать site.
2. Создать upstream.
3. Привязать TLS при необходимости.
4. Скомпилировать revision.
5. Применить revision.
6. Проверить `/healthcheck` и `Requests`.

## Безопасно выкатить большой пакет изменений

1. Сохранить изменения с `X-WAF-Auto-Apply-Disabled: true`.
2. Проверить ожидаемое состояние.
3. Скомпилировать один раз.
4. Применить один раз.
5. Проверить `Dashboard`, `Events` и metrics.

## Аккуратно ужесточить CRS

1. Зафиксировать текущий revision.
2. Изменить CRS-related policy.
3. Выполнить compile/apply.
4. Следить за false positives в `Requests` и `Events`.
5. Добавлять exclusions только там, где это действительно нужно.

## Разобрать заблокированный запрос

1. Найти запрос в `Requests`.
2. Связать его с `Events`.
3. Понять, кто блокировал: WAF, rate-limit или Anti-DDoS.
4. Решить, нужен ли policy tuning или блокировка корректна.

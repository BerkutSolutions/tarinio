# Contributing

Product: **Berkut Solutions - TARINIO**

## Scope and principles

- Keep runtime fully offline (no external CDN/runtime downloads).
- Keep RU/EN documentation and i18n in sync.
- Keep changes minimal and consistent with existing patterns.

## Development workflow

1. Run tests (`go test -count=1 ./...`).
2. Update documentation when behavior changes.
3. Update `CHANGELOG.md` for user-visible changes.

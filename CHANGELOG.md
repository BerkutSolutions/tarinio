# Журнал изменений

Все значимые изменения проекта фиксируются в этом файле.

## [1.1.21] - 2026-04-18

### Установка / AIO
- Доработана очистка терминала в `install-aio.sh` для запуска через `curl ... | sh`: к ANSI-reset и `tput` добавлен жёсткий fallback с прокруткой экрана и повторной очисткой, чтобы стартовые строки оболочки не оставались видимыми в терминалах с частичной поддержкой `clear`.

### Dashboard / CPU
- Системная метрика `Нагрузка CPU` теперь считается по фактической загрузке из `/proc/stat`, а не по `loadavg`, поэтому значения стали ближе к реальной утилизации ВМ.
- CPU контейнеров в dashboard нормализован по числу ядер, чтобы суммарный процент и пер-контейнерные проценты не выглядели завышенными относительно реальной загрузки сервера.

### UI / Layout
- Усилена глобальная защита от переполнения текста в карточках, модалках, контейнерных виджетах и системных метриках dashboard: длинные строки и большие значения теперь переносятся внутри фрейма.

### Версия
- Версия приложения обновлена до `1.1.21`.

## [1.1.20] - 2026-04-18

### Healthcheck / Probes
- Исправлены healthcheck-пробы `API дашборда`, `API запросов` и `API событий`: они переведены на lightweight probe-режим через `/api/dashboard/stats?probe=...`, без запроса тяжёлых payload и без пересечения со special-case роутом UI nginx для `/api/requests`.
- Медленные ответы (`needs_attention`) теперь показываются warning-статусом, а не серым neutral-state.

### Runtime / Nginx
- Дополнительно увеличены `variables_hash_max_size` и `variables_hash_bucket_size`, чтобы убрать warning `could not build optimal variables_hash` на инстансах с большим числом переменных.

### Live Logs
- Для live logs контейнеров добавлена дедупликация строк по `timestamp + message`, чтобы при polling не появлялись повторяющиеся записи одной и той же строки.

### Установка / AIO
- Усилена очистка терминала в `install-aio.sh`: перед выводом теперь выполняется полный terminal reset (`ESC c`) с дополнительной очисткой экрана и scrollback.

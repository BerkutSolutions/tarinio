## [2.0.7] - 23.04.2026

### Установка и первый запуск
- Исправлен upgrade installer для существующих standalone-инсталляций: сохранение старых секретов, нормализация `POSTGRES_DSN`, корректное завершение footer-блока без ложного `[FAIL]`.
- Исправлен onboarding standalone-контура: ACME `http-01` теперь выпускается после первого `compile/apply`, а переход на HTTPS больше не ломает финальную проверку статуса.
- Исправлено первое применение runtime поверх bootstrap-конфига: relink корректно заменяет непустые каталоги, а ошибки `/reload` теперь возвращают содержимое тела ответа.
- Маршруты `/api/*`, `/login`, `/onboarding/*` и `/static/*` сохранены доступными во время onboarding после первого apply, чтобы браузер не падал на `Failed to fetch`.

### Импорт сервисов
- Импорт `.env` и `.json` файлов переведён в staged-режим без промежуточного auto-apply на каждом `POST/PUT`.
- После завершения импорта теперь выполняется один финальный `compile/apply`, поэтому импорт больше не создаёт каскад ревизий и не шумит промежуточными ошибками на частично собранных ресурсах.
- Проверка существующих site/upstream/tls-ресурсов при импорте переведена на инвентарь из списков, без лишних `404`-probe запросов поштучно.

### Диагностика runtime
- Ошибки `waf-runtime-l4-guard` теперь поднимаются с реальным stderr helper-а, а не только как `exit status 1`.
- Сообщения о failed apply в onboarding теперь показывают фактическую причину failed-job, а не общий `Initial apply failed`.

### Версия и документация
- Версия продукта, UI, release tests и release-synced documentation обновлены до `2.0.7`.
- `CHANGELOG.md` сокращён до компактного релизного summary вместо перегруженного списка.

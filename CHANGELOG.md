## [1.2.5] - 24.06.2026

### UI
- Исправлена детализация атак на дашборде: при наличии security events счётчики и breakdown больше не подменяются массовыми `403/429` из request-логов.
- Автообновление дашборда теперь гарантированно снимает фоновые таймеры при уходе со страницы, а время обновления дневных индексов отображается локализованно.
- Для таймаутов management API UI снова показывает фирменную fallback-страницу WAF вместо стандартной `nginx 504 Gateway Time-out`.
- Для easy-site логина challenge теперь проходит до auth-gate логина, а не после первой неуспешной попытки входа.
- Детализация виджетов дашборда теперь кэшируется на короткое окно и при кратком сбое `/api/requests` или `/api/events` удерживает последнюю валидную сводку вместо провала в пустые таблицы и `0 unique IP`.
- Автообновление дашборда снижено до более щадящего интервала, а обзор контейнеров теперь догружается отдельно и больше не тормозит основной ответ `/api/dashboard/stats`.
### Ядро
- Исправлен `healthcheck /healthcheck`: probe запросов больше не валит API дашборда ответом `502`, если runtime request telemetry временно недоступен или деградировал.
- Для management-site маршруты `/api/*` теперь проксируются в `control-plane:8080`, а не в UI upstream. Это убирает `504` и `upstream timed out` для `/api/dashboard/stats`.
- Логика определения management-site теперь учитывает и кастомный `CONTROL_PLANE_DEV_FAST_START_MANAGEMENT_SITE_ID`, а не только жёстко заданные идентификаторы.
- Дашборд больше не считает WAF challenge-страницы с query-string как обычные клиентские запросы, а при временном сбое request telemetry использует последнюю успешную сводку вместо резкого падения счётчиков в `0`.

### Безопасность
- Убран шум от runtime-запросов к индексам хранения: control-plane теперь передаёт `X-WAF-Runtime-Token` при обращении к `/requests/indexes`.
- Добавлены тесты на передачу runtime token в fetch/delete запросах к индексам.
- Глобальный anti-DDoS L7 override больше не применяется к management-site даже при кастомном ID, чтобы не ломать админский контур.

### Логирование
- Исправлена загрузка дневных индексов в настройках логирования: данные больше не обнуляются из-за отсутствующего runtime token.
- Заголовок секции холодного слоя и статусные тексты отвязаны от `ClickHouse` и теперь описывают именно холодное хранилище.
- Адреса `OpenSearch`, `ClickHouse` и `Vault` теперь подставляются автоматически по выбранному backend и данным о запущенных контейнерах стека, с безопасным fallback на стандартные внутренние адреса.
- Для секретных полей логирования и Vault добавлены переключатели показать/скрыть.
- В Администрирование добавлен новый скрипт «Сборщик здоровья индексов WAF» для диагностики `/requests/indexes`, probe-эндпоинтов, OpenSearch и ClickHouse в одном архиве.

### UI
- Фоновый polling дашборда теперь не идёт во скрытой вкладке и не запускает новый запрос, пока предыдущий ещё не завершился. Это снижает шанс словить лишние `504` и уменьшает шум в логах.

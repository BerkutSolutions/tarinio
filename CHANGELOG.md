# Журнал изменений

Все значимые изменения проекта фиксируются в этом файле.

## [1.1.0] - 2026-04-06

### Версионирование
- Версия приложения и документации поднята до `1.1.0`.

### UI / Dashboard
- В деталях виджета `Заблокированные атаки` добавлен список атакующих IP с количеством блокировок и переходом в IP-детали по клику.

### UI / Bans
- Исправлен источник данных страницы `Баны`: теперь список строится не только по ручному `access_policies.denylist`, но и по автоматическим блокировкам из `security_access`/`security_rate_limit` событий.
- Добавлена агрегация авто-блокировок по `site + ip` с расчетом времени действия (TTL) на основе `bad_behavior_ban_time_seconds` easy-profile.
- Добавлен просмотр подробностей по ЛКМ на строке бана: причина, статусы, количество срабатываний, связанные пути/хосты/referer/user-agent и сырой payload последнего события.
- Для авто-блокировок отключено действие `Разбанить` в таблице (ручной `unban` применим только к manual denylist), чтобы исключить ложное ожидание снятия runtime-блокировки через denylist API.
- Добавлена кнопка `Забанить` на вкладке `Баны` с формой `сайт + IP + время (сек)`; для заданного времени реализован автоматический разбан при следующем открытии/обновлении вкладки.

### I18N / RU
- Исправлены артефакты кодировки в `docs/ru/README.md`, файл переведен в корректный UTF-8 текст.

## [1.0.19] - 2026-04-06

### Версионирование
- Версия приложения и документации поднята до `1.0.19`.

### UI / Dashboard
- Исправлен виджет `Уникальные IP атакующих`: теперь IP отображается рядом со страной и флагом.
- Улучшена нормализация кода страны (включая формат `ru (RU)`), чтобы корректно отображались эмодзи-флаги.
- В виджете здоровья контейнеров исключены нерелевантные контейнеры вне docker compose-проекта (например, `happy_booth`/`wonderful_einstein`).

### TLS / Certificates
- Добавлен экспорт существующих сертификатных материалов через API: `GET /api/certificate-materials/export/{certificate_id}`.
- Добавлен массовый экспорт сертификатных материалов в архив: `POST /api/certificate-materials/export`.
- Добавлен API глобальных настроек автообновления сертификатов: `GET /api/tls/auto-renew`, `PUT /api/tls/auto-renew`.
- Добавлена backend-логика фонового автообновления сертификатов по порогу дней до истечения срока.
- Для сохранения/редактирования сервиса устранен лишний перевыпуск сертификатов: повторный ACME/self-signed выпуск выполняется только если сертификат еще не существует.
- В мастере сервиса (`Sites`) для `Сервер центра сертификации` добавлен режим `Импорт`.
- Добавлены кнопки `Импорт` и `Экспорт` материалов сертификата рядом с полем `Certificate ID`.
- Для режима `Импорт` сохранение сервиса использует уже импортированный сертификат как основу и не запускает ACME-выпуск.

### UI / TLS
- На вкладке `TLS / Сертификаты` добавлен блок `Сертификаты` с выбором строк и действиями `Импорт`/`Экспорт`.
- Реализован экспорт выбранных сертификатов: один выбранный сертификат экспортируется парой PEM-файлов, несколько сертификатов экспортируются архивом.
- Добавлен блок `Автообновление сертификатов` с чекбоксом, полем дней до истечения и явной кнопкой `Сохранить`.

### I18N / RU
- Точечно исправлены непереведенные и некорректные русские строки в разделах `TLS`/`Services`/`Onboarding`.
- Исправлена формулировка `Сертификат authority server` на `Сервер центра сертификации`.

## [1.0.17] - 2026-04-05

### Версионирование
- Версия приложения и документации поднята до `1.0.17`.

### Рантайм / NGINX / GeoIP
- Исправлен старт `runtime` при включенных GeoIP-директивах: launcher теперь загружает оба необходимых динамических модуля NGINX (`ngx_http_modsecurity_*` и `ngx_http_geoip_*`) до запуска nginx.
- Устранена причина падения `runtime` с ошибкой `unknown directive "geoip_country"` в `base.conf`, из-за которой порты `80/443` не поднимались и ACME HTTP-01 в онбординге завершался `connection refused`.
- Синтакс-валидация компилятора (`RuntimeSyntaxRunner`) дополнена загрузкой GeoIP-модуля (когда он доступен), чтобы такие проблемы ловились до применения ревизии.
- Исправлен краш `runtime` из-за отсутствующей линковки каталога `nginx/easy-locations` в launcher-подготовке layout.
- Исправлен краш `runtime` с `duplicate location "/api/"`: генерация route-specific easy-limit локаций теперь пропускает зарезервированные пути `/` и `/api/`.
- Добавлен regression-тест для reserved путей в генерации easy-limit артефактов.

### Auto-start / Localhost TLS
- В `dev fast start` для loopback/`localhost` хоста включен принудительный self-signed выпуск сертификата (без ACME-челленджа), чтобы `https://localhost/login` стабильно поднимался после `docker compose up -d --build` в `deploy/compose/auto-start`.
- Добавлен unit-тест `TestShouldUseSelfSignedForDevFastStartHost` (`control-plane/internal/app/dev_fast_start_tls_test.go`) для проверки выбора self-signed issuer на localhost/loopback.

### UI / Дашборд
- Исправлена стилизация выпадающего списка размера страницы на вкладках `Активность`, `События` и `Запросы`.
- Переименована секция `Логи / Активность` в `Активность` (RU/EN локали).
- Убраны дублирующие внутренние заголовки карточек на вкладках `Активность`, `События` и `Запросы`.
- В виджете стран на дашборде добавлен явный показ флага рядом с кодом страны.

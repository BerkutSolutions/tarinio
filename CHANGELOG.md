## [1.5.3] - 15.07.2026

### WAF, Nginx и страницы ошибок

- Добавлена общая политика ограничения прямого IP-доступа в «Настройки → Безопасность». При выключенной политике неизвестный Host или прямой IP получает брендированную страницу `421`; при включённой nginx закрывает соединение кодом `444`. Настройка аудируется и после сохранения автоматически компилирует и применяет ревизию.
- HTTP- и HTTPS-серверы по умолчанию классифицируют неизвестный Host и прямой IP, записывают нормализованную причину безопасности в журнал запросов и используют общие страницы ошибок WAF вместо стандартной страницы nginx.
- HTTP-01 ACME для нового домена освобождён от обработки неизвестного Host, ModSecurity и кастомных страниц ошибок: путь `/.well-known/acme-challenge/` корректно выдаёт токен до появления домена в активной карте хостов.
- Исправлены два дефекта генерации nginx: регулярное выражение IPv4 больше не использует несовместимые с nginx квантификаторы в фигурных скобках, а `limit_req_status` на маршрутах входа панели не объявляется повторно.
- Кастомные страницы ошибок больше не заменяют ответы upstream management API HTML-разметкой. `/api/` сохраняет оригинальные статус и тело ответа, при этом WAF, anti-bot, CRS и лимиты продолжают применяться.
- Настройки отдельных кастомных страниц ошибок корректно исключают только выбранные статусы; внутренние пути `__waf_errors` недоступны извне. Для всех HTTP-статусов и Geo Block добавлены переводы `ru`, `en`, `de`, `sr` и `zh`.

### Защита панели управления и сервисов

- Management host теперь проходит те же anti-bot, лимиты запросов и подключений и CRS, что и обычные сервисы; безопасные маршруты challenge и статических ресурсов остаются исключениями, поэтому CSS, JavaScript и логотип страницы входа не попадают в цикл перенаправления.
- `/login` и `/login/2fa` защищены до рендеринга формы; вход с отсутствующим passkey возвращает штатный локализуемый JSON-результат вместо страницы `404` или `400` WAF.
- Выделены и проверены management-маршруты API, в том числе при конфигурации management host через окружение. Обычные сервисы не получают эти исключения автоматически.
- Исправлена ложная блокировка preflight при применении ревизии: bypass ModSecurity для management host требуется только когда ModSecurity действительно включён в его nginx-маршруте.
- Дополнены E2E-сценарии создания и изменения сервисов через management host, а также сценарий L7-флуда, эскалации адаптивной модели до `drop` и L4-блокировки IP атакующего.

### Мониторинг, запросы и Anti-DDoS

- Виджет за 24 часа переименован в «Запросы и атаки во времени (24ч)», стал на один шаг сетки уже и мигрирует ранее сохранённую ширину. В нём раздельно отображаются почасовые серии запросов и атак с точными значениями при наведении.
- Сумма `attacks_series` синхронизирована с дневным количеством атак: журналы запросов используются как fallback только при отсутствии событий WAF.
- Добавлен локальный генератор телеметрии для тестового стенда с реалистичными запросами, атаками и разными почасовыми пиками. Страны показываются из локального набора PNG-флагов `16×12`, без CDN и emoji-заглушек.
- Обновлены графики панели и страницы запросов: убраны лишние базовые линии, улучшены подписи шкал, данные при наведении, причины безопасности и отображение прямого IP или неизвестного Host.

### Healthcheck, вход и интерфейс

- `/healthcheck` получил пять сохраняемых runtime-вариантов внешнего вида с предпросмотром и живыми проверками сессии, API, совместимости и контейнерных ошибок. Добавлены публичные API для выбранного варианта healthcheck и страницы входа с серверной валидацией значений.
- Во всех вариантах healthcheck длинные ошибки переносятся по строкам и не расширяют фрейм или страницу; свёрнутые аккордеоны сохраняют компактную высоту. Для Security Assurance добавлена широкая консоль диагностики, а Status Board по умолчанию оставляет группы свёрнутыми и сохраняет сообщения успешных проверок.
- Кнопки обновления и перехода в панель приведены к компактному стилю высотой `44px`.
- Страницы `/login` и `/login/2fa` используют единый каркас и пять локализованных тем. Убраны legacy-рендер, лишний фоновый `/api/auth/me`, устаревшая тема Sidebar Rail и несогласованное положение элементов управления в Incident Console.
- Боковое меню перегруппировано в Monitoring, Management, Protection и System без изменения RBAC или маршрутов; добавлены статус runtime, активная ревизия, CRS и индикаторы L4/L7/MODEL.
- Боковое меню и его элементы управления переведены на локальные SVG `32×32`; значки профиля, уведомлений, сворачивания и выхода имеют единый размер и состояние, а значок выхода центрирован.
- Обновлены логотипы, favicon и скриншоты интерфейса; удалён устаревший asset `logo500x300.png`.

### Тесты и качество

- Добавлены compiler-регрессии для ACME HTTP-01, брендированной `421`, политики прямого IP, management API, anti-bot маршрутов входа, выборочных кастомных страниц ошибок и внутренних error-artifacts.
- Дополнены UI-contract, i18n, layout dashboard, внешний вид healthcheck и error-pages тесты, а также runtime/E2E-сценарии для сервисов, ModSecurity, Geo Block и Anti-DDoS.
- Smoke и quality-matrix E2E для `/healthcheck` проверяют актуальный runtime-shell `healthcheck-app` и загрузку его модуля вместо удалённых маркеров прежней статической вёрстки.
- Усилен runtime syntax pipeline: сгенерированные артефакты проверяются перед применением ревизии, что предотвращает перезапуск runtime из-за некорректной конфигурации nginx.

### Релиз, документация и инфраструктура

- Версия продукта, документации, fallback-маркеров UI и release-навигации синхронизирована на `1.5.3`; скрипт выпуска теперь обновляет также legacy-версию в runtime shell.
- Go и все Docker builder-образы обновлены до `1.26.5`, включая контрольную плоскость, runtime, CLI и compose-профили.
- В testpage compose добавлен отдельный контейнер `e2e-attacker` для изолированных проверок защиты. README и сайт документации используют актуальные версию и логотип.

## [1.2.7] - 25.06.2026

### UI
- В редакторе сервиса добавлен новый блок исключений anti-bot по URL и HTTP-методам. Он расположен над двухслойной эскалацией challenge и позволяет безопасно выводить из challenge API и машинные эндпоинты вроде `/api/`, webhook-путей и healthcheck-ов.
- Исключения anti-bot теперь проходят полный цикл через easy-редактор, raw env, повторное открытие карточки сервиса, импорт и экспорт конфигураций без потери значений.
- Для новых полей и ошибок добавлены локализации `ru`, `en`, `zh`, `de`, `sr`.

### Ядро
- В контракт easy-site profile добавлены `security_antibot.exclusion_rules` с путём и списком HTTP-методов.
- Исключения anti-bot теперь нормализуются, валидируются и передаются из control-plane в compiler и runtime без ручных обходных сценариев.
- В режимах `monitor` и `transparent` anti-bot исключения автоматически отключаются вместе с challenge, чтобы не появлялись неактуальные правила.

### Runtime
- Easy nginx-конфигурация теперь умеет пропускать запросы по явно заданным anti-bot исключениям до применения challenge и двухслойной эскалации.
- Это позволяет не ломать браузерный challenge на сайте и одновременно не резать API-трафик, SDK, webhook-и и служебные URL.

### Тесты
- Добавлен compiler-тест на генерацию anti-bot исключений по URL и методам.
- Обновлены contract/round-trip тесты easy-site profile для нового поля `exclusion_rules`.

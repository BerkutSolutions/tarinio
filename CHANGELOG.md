# Журнал изменений

Все значимые изменения проекта фиксируются в этом файле.

## [1.1.17] - 2026-04-18

### Администрирование / Скрипты
- На вкладку `Администрирование` добавлен новый блок `Скрипты` с отдельными карточками запуска для диагностических сценариев.
- Для каждого скрипта добавлены отдельные параметры ввода, локализованные подписи полей, статус выполнения и ссылка на скачивание собранного архива.

### Диагностика / Сбор ошибок
- Добавлен новый скрипт `scripts/collect-waf-errors.sh` для выгрузки runtime/control-plane/ui/ddos-model ошибок, warning-сообщений и сводок без интерактивной ручной обработки логов.
- `scripts/collect-waf-events.sh` переведён в более удобный режим автоматизации: поддерживает `NON_INTERACTIVE`, входные значения через env, локальный `waf-cli` и настраиваемую директорию выгрузки.

### Healthcheck / Логи
- В `Healthcheck` добавлен отдельный accordion-блок с уникальными проблемами из логов контейнеров.
- Повторы дедуплицируются по нормализованному сообщению без учёта времени, при этом в UI сохраняются корректные `first seen`, `last seen`, `count` и оригинальный пример строки лога.
- Для warning-сообщений добавлен отдельный жёлтый статус вместо серого neutral-state.
- Из блока исключён заведомо безопасный Redis startup warning про отсутствие явного `redis.conf`, чтобы он не засорял диагностику.

### UI / Лайв-логи
- Исправлен баг, из-за которого при автообновлении live logs скролл сбрасывался вверх и мешал нормально просматривать новые записи.

### Runtime / Шум в логах
- Снижен спам в `control-plane` от `runtime security collector failed` за счёт более аккуратного throttling/backoff.
- Уменьшен шум nginx/runtime-логов: выставлен `error_log warn`, добавлены `variables_hash_*`, а для `/.well-known/acme-challenge/` отключён `log_not_found`.

# Журнал изменений

Все значимые изменения проекта фиксируются в этом файле.

## [1.1.8] - 2026-04-07

### Ядро / Исключения IP
- Добавлен компиляторный bypass для `allowlist` (IP и CIDR): адреса из исключений теперь обходят `limit_req/limit_conn`, включая route-specific easy-лимиты.
- Добавлен guard в easy-конфиге: для адресов из исключений не срабатывают country-ограничения (`whitelist/blacklist`) и antibot/rate-limit блокировки.

### Интерфейс / Баны
- Исправлен расчет времени автобанов в UI: если у сервиса задан `whitelist_country`, записи в таблице банов отображаются как перманентные (вместо ложного `Истек` по таймеру `bad_behavior_ban_time_seconds`).
- Кнопка `Разбанить` активирована и для автоматических банов: теперь действие добавляет IP в исключения (allowlist) сервиса, чтобы трафик с этого адреса больше не блокировался лимитами.

### Интерфейс / Сервисы
- В редакторе сервиса в блоке списков добавлен переключатель `Включить исключения`.
- Рядом с `Разрешенными` добавлено отдельное поле `Исключения` (IP/CIDR), применяемое как bypass для limit/access-country блокировок.
- В селекторах стран формат обновлен на человекочитаемый: `Полное название (🇺🇸)` без буквенного country-кода в скобках.

### Интерфейс / Дашборд и Баны
- В виджетах стран отображаются полные названия стран с emoji-флагом в скобках.
- Для country-кодов исправлена нормализация (`ru`/`"ru"`/`RU` -> `RU`), чтобы флаг рендерился как emoji.
- В таблице и деталях банов вместо кода страны показывается только emoji-флаг.

### Runtime / Error Pages
- Для runtime nginx добавлен глобальный стилизованный fallback `errors/_global/*` и привязка `error_page` в default HTTP server для расширенного набора кодов (`4xx/5xx`, включая `421`, `429`, `520-526`), чтобы не показывались “белые” страницы nginx.
- Каталог стилизованных error-страниц расширен: страницы теперь генерируются не только для базовых кодов, но и для полного поддерживаемого набора ошибок WAF/runtime.

### Runtime / HTTP
- В default HTTP server (`listen 80 default_server`) вместо `421` добавлен принудительный редирект `308 -> https://$host$request_uri` (исключая ACME challenge path), чтобы после настройки TLS не оставалась страница `421 Misdirected Request` на `http://...`.

### API / Easy Profile
- В `security_behavior_and_limits` добавлены поля `use_exceptions` и `exceptions_ip`.
- Синхронизация easy-profile -> access policy обновлена: исключения сохраняются в `allowlist`, а при чтении legacy policy автоматически подтягиваются обратно в easy профиль.

### Тесты
- Добавлены/обновлены тесты компилятора для проверки bypass-ключей (`geo/map`) и новых guard-правил в easy/rate-limit шаблонах.
- Добавлены/обновлены тесты `easysiteprofiles` и сервисного моста для полей исключений.

## [1.1.7] - 2026-04-06

### Интерфейс / Баны
- Убраны из UI служебные `startup-self-test-*` записи в списке банов и в выборе сайта для ручного бана.
- Для действия `Разбанить` добавлено полноценное модальное окно подтверждения (вместо браузерного `confirm`).

### API / Ручной бан
- Исправлен `unban` для случаев, когда сайт уже удален, но policy еще существует: операция теперь выполняется по найденной policy и не падает `404 site ... not found`.

### Startup Self-Test
- Добавлена очистка устаревших `startup-self-test-*` access policy при старте.
- При завершении startup self-test дополнительно удаляется связанная access policy (`<site>-access`), чтобы не оставались “лишние” баны на проде.

## [1.1.6] - 2026-04-06

### Интерфейс / Баны
- В модальном окне `Забанить` в поле `Сайт` добавлен пункт `Все сервисы`.
- При выборе `Все сервисы` ручной бан применяется сразу ко всем сервисам (массовый `ban` по сетке сервисов).

### API / Ручной бан
- Для `ban/unban` добавлен серверный режим `all-services` (`*`, `all`, `__all__`): одна операция применяется ко всем сервисам.
- Добавлены тесты сервиса ручных банов на массовый `ban/unban` по всем сервисам.

### Безопасность / Принудительные рантайм-блокировки
- Подтверждено и зафиксировано поведение перманентной блокировки на runtime:
  - `security_behavior_and_limits.blacklist_ip` компилируется в `deny`-правила nginx;
  - `security_country_policy.whitelist_country` и `blacklist_country` компилируются в жесткие country-гейты (`403`).
- Эти правила действуют постоянно, пока присутствуют в конфигурации (не зависят от таймеров UI).

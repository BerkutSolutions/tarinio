# Журнал изменений

Все значимые изменения проекта фиксируются в этом файле.

## [2.0.1] - 2026-04-19

### UI / Runtime
- В редакторе создания и редактирования сервиса сохранён основной рабочий `card`, а лишние внутренние tab-рамки убраны, чтобы форма не выглядела как двойной фрейм.
- Runtime security events больше не смешивают трафик разных сайтов в один burst-key, игнорируют статические ассеты и используют более реалистичные пороги по умолчанию, чтобы обычная авторизованная навигация не засоряла событиями `request burst detected (not blocked)`.
- Для `login` и runtime `429`-страницы обновлён fallback/blocked UI, чтобы страница ограничения снова выглядела как штатный стилизованный экран, а не как грубая техническая заглушка.
- В боковом меню визуальный акцент перенесён на иконку, чтобы текст пунктов не сидел внутри тяжёлого «облака».
- В администрировании добавлены i18n-подписи для стандартных ролей и permission-кодов, чтобы списки разрешений и выбор ролей отображались человекочитаемо на русском и английском.
- Добавлены локальные интерфейсные языки `de`, `sr`, `zh`, сохранение выбранного языка между перезагрузками и browser-language fallback на английский для неподдерживаемых локалей.
- Статусные и blocked/fallback-страницы (`index`, `login`, `login-2fa`, `onboarding`, runtime `429`) теперь тоже выбирают язык по браузеру и имеют локальные переводы без внешних CDN/API.
- Добавлена проверка дополнительных locale-файлов в UI-тестах: `de/sr/zh` теперь обязаны быть синхронны по ключам с `en` и не содержать битых строк.
- Добавлены новые языки интерфейса `Китайский`, `Сербский`, `Немецкий`, а автоопределение языка для интерфейса и статусных/blocked-страниц теперь использует язык браузера с fallback на английский.
- Исправлено сохранение пользовательской локали: выбранный язык теперь не теряется после перезагрузки и имеет приоритет над автоопределением по браузеру.

### Баны / UI
- Ручной `Разбан` по кнопке теперь переживает перезагрузку страницы для строк, собранных из runtime-событий: UI сохраняет серверный `accesspolicy.unban`-маркер и больше не восстанавливает старую autoban-строку из исторических событий, если разбан уже был подтверждён позже.

### Версия / Документация
- Проект, UI fallback-строки и release-synced wiki синхронизированы на версию `2.0.1`.

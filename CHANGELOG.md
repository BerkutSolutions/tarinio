## [1.5.1] - 13.07.2026

### Error-page visual design

- Страницы 451, 494–497, 499, 506 и 520–526 получили самостоятельные заголовки, подробные пояснения причин, акцентные цвета и эмблемы вместо переиспользования общего текста 400/500.
- Для каждого расширенного кода добавлены уникальные путь запроса, состояние стрелок и действия по восстановлению: визуально выделяется фактическая точка сбоя у клиента, WAF, сети, TLS или origin-сервера.
- Конечный узел маршрута унифицирован как `Клиент → WAF → Сервер` в русской локали и `Client → WAF → Origin` в английской; это применяется и к legacy preview-страницам, и к runtime-артефактам.
- Значок конечного сервера теперь отражает состояние маршрута: используются нейтральный сервисный значок при доступном или не достигнутом сервере и цветные состояния для тайм-аутов, TLS и сертификата; крест остаётся для реальной недоступности сервера.
- Обновлены эмблемы 495, 522, 524, 525 и 526: скорректировано положение элементов, добавлен красный центральный разрыв TLS-рукопожатия и отдельные символы для тайм-аутов и сертификата.
- Локализованы русские подписи маршрута, состояния и инструкции по восстановлению; общая локализация страницы больше не перезаписывает кодоспецифичный текст.

### Страницы ошибок WAF

- Добавлены страницы 494–497, 499, 506 и 520–526; они доступны в списке, preview API и runtime-артефактах WAF.
- `451 Geo Block` разделена на `451 Unavailable For Legal Reasons` и отдельную `Geo Block` (HTTP 403). Геополитика использует внутренний маркер и тело `geo_block`, не подменяя обычную страницу 403.
- Старое отключение страницы `451` автоматически распространяется на `geo_block`, без дубликатов; 451 остаётся совместимым идентификатором юридической страницы.
- В UI страницы сгруппированы по источнику ошибки; коды Nginx/proxy и upstream/CDN явно помечены как vendor-specific, а 499 — как диагностический.
- Preview и runtime-страницы 494–497, 499, 506 и 520–526 теперь имеют собственные код, название и локализованное объяснение вместо переиспользования текста 400/500; в 451 восстановлена ссылка Tarinio в подвале.

### Ядро

- Единственный активный внутренний upstream `ui:80` временно распознаётся как управляющий до сохранения явного списка хостов, чтобы обновлённая панель не блокировала собственную настройку.
- Переименование сервиса теперь мигрирует связанные upstream, TLS-привязку, сертификатный материал, easy-профиль, политики, virtual patches и management-host, после чего автоматически применяет одну итоговую ревизию.
- Повторное сохранение уже перенесённого сервиса не перевыпускает и не изменяет его сертификат.

### Nginx

- HTTPS-запрос к отключённому или неизвестному хосту теперь получает фирменную страницу 421 вместо стандартного ответа nginx.
- Default HTTPS server использует фирменные WAF-страницы для всех поддерживаемых nginx error codes.
- Удалён legacy fallback `status.html.tmpl`: production-артефакты страниц ошибок формируются только из локализованных WAF preview-шаблонов; отсутствие шаблона теперь останавливает компиляцию.

### Телеметрия

- В телеметрию попадают браузерные запросы к management-host, включая `https://localhost` в default compose. Исключаются только служебные обращения из контейнеров к `control-plane` и внутреннему loopback, чтобы технический трафик не искажал метрики.

### Интерфейс и API

- В редакторе страниц ошибок коды сгруппированы по источнику сбоя; добавлены отдельные локализованные подписи, маркеры vendor-specific и диагностический маркер для 499.
- API обновления сайта поддерживает безопасное переименование ID сервиса с миграцией связанных upstream, TLS, сертификатных материалов, профилей и политик; после миграции автоматически применяется итоговая ревизия.
- В preview API и runtime-генерации страниц ошибок добавлена единая нормализация названия конечного узла маршрута.

### Проверки

- Основной E2E-контракт Geo Block приведён к публичному HTTP 403; юридический HTTP 451 проверяется независимо.
- Обновлены unit- и контрактные тесты генерации страниц ошибок, миграции сервисов, telemetry filters и конфигурации Nginx.
- Выполнены полный `go test ./...` и smoke E2E-сценарий входа, healthcheck и dashboard.

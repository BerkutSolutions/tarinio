# Журнал изменений

Все значимые изменения проекта фиксируются в этом файле.

## [1.1.14] - 2026-04-15

### Easy / Blacklist URI
- Добавлена безопасная поддержка wildcard-форматов в `security_behavior_and_limits.blacklist_uri` (например, `*.php`).
- Если значение `blacklist_uri` не является валидным regex, оно автоматически интерпретируется как wildcard-шаблон (`*`/`?`) с экранированием спецсимволов, чтобы не ломать nginx-конфиг.

### Easy / Custom rate limits
- Кастомные правила лимитов в `custom_limit_rules` теперь возвращают тот же класс кодов, что и политика bad behavior: код выбирается из включённых `bad_behavior_status_codes` (приоритет `429 -> 403 -> 444`, fallback `429`).
- Для кастомных лимитов добавлен явный `limit_req_status` в generated nginx locations, чтобы не отдавать дефолтный `503` от `limit_req`.
- Для Sentry-подобных endpoint'ов добавлена совместимость по wildcard-пути: при правиле вида `/api/<id>/envelope/` автоматически добавляется alias `/api/*/envelope/`, чтобы события не терялись из-за смены project id.

### Easy / Access policy
- Исправлен сценарий редактирования сервиса с очисткой allow/deny списков: удаление access-policy стало устойчивым к legacy-id и авто-apply ошибкам (`400`), с повторной проверкой фактического удаления по `site_id`.

### Easy / Traffic control UX
- Расширены встроенные quick templates для `blacklist_uri` и `blacklist_user_agent` (добавлены дополнительные наборы для probe/debug/backup/webshell/fuzzer/exploit-паттернов).
- Добавлены недостающие i18n-ключи на вкладке `Контроль трафика`: локализованы подписи quick templates, кнопка добавления шаблона, поиск стран и заголовок `Bad Behavior` в RU.

### Диагностика / Сбор инцидентов
- Добавлен интерактивный скрипт `scripts/collect-waf-events.sh` для быстрого сбора данных по событиям WAF:
  - логин/пароль CLI;
  - фильтры по IP (опционально), сервису/host (опционально), коду ответа (опционально);
  - выгрузка `events`, `access-policies`, `antiddos`, `bans`, фрагментов runtime/control-plane логов и nginx-фокуса;
  - упаковка результата в архив для последующего анализа.

### Версия
- Версия приложения обновлена до `1.1.14` в backend metadata и пользовательских точках отображения UI.

## [1.1.13] - 2026-04-08

### Запросы / Прод-стабильность
- Снижен риск «зависания» вкладки `Запросы` на инстансах с большим `access.log`: ingestion в runtime переведён на пакетную обработку (батчами), без длительной блокировки одного запроса.
- В `settings/runtime` отключён лишний вызов индексов при обычном чтении настроек (индексы запрашиваются только при явных `storage_indexes_*` параметрах), чтобы не замедлять связанные страницы и проверки.

### Healthcheck / Диагностика
- Улучшена проверка `API запросов`: probe теперь выполняется в «лёгком» режиме (`limit=1` + текущий день), чтобы исключить долгие прогрузы на больших объёмах исторических данных.
- Добавлены таймаут probe и явная пометка медленных ответов в healthcheck (вместо долгого «Проверка...» без статуса).
- В раздел `Совместимость` добавлен агрегированный индикатор деградаций runtime API probes (без ложной кнопки `Исправить` для диагностической записи).
- Обновлён контракт совместимости до `2026-04-08-healthcheck-v2` (UI, backend registry, installer).

### Login / Интерфейс
- Убран лишний pre-check `/api/auth/me` на страницах `login` и `login-2fa`, чтобы не показывать ожидаемую `401 Unauthorized` ошибку до авторизации.

### Startup Self-test / Prod hygiene
- Усилен startup self-test: теперь проверяется полный lifecycle сервиса (`create -> update -> delete`) для `site/upstream/tls/easy profile` с валидацией, что после удаления не остаётся доменных следов в snapshot.
- Для self-test отключены audit side-effects через context, чтобы тест не добавлял записи в `audits`.
- Компиляция для self-test переведена в режим preview (без записи `revisions/jobs/revision-snapshots`), чтобы не засорять persistent storage.
- Добавлена финальная проверка `assertNoStartupSelfTestArtifacts`: после self-test в рабочих storage не должно быть `startup-self-test-*` сущностей, включая тестовые сертификаты.
- Добавлена очистка stale-хвостов `startup-self-test-*` перед запуском self-test (site/upstream/tls/profile/policies/certs).

### Интерфейс / Запросы
- Фильтры `Сервис`, `Метод`, `Статус`, `Время` размещены в первой строке.
- Для выпадающих списков на вкладке `Запросы` добавлены ограничения ширины и фиксированная сетка, чтобы поля не разъезжались по ширине при выборе разных значений.

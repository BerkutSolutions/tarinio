# Журнал изменений

Все значимые изменения проекта фиксируются в этом файле.

## [1.1.13] - 2026-04-08

### Запросы / Прод-стабильность
- Снижен риск «зависания» вкладки `Запросы` на инстансах с большим `access.log`: ingestion в runtime переведён на пакетную обработку (батчами), без длительной блокировки одного запроса.
- В `settings/runtime` отключён лишний вызов индексов при обычном чтении настроек (индексы запрашиваются только при явных `storage_indexes_*` параметрах), чтобы не замедлять связанные страницы и проверки.

### Healthcheck / Диагностика
- Улучшена проверка `API запросов`: probe теперь выполняется в «лёгком» режиме (`limit=1` + текущий день), чтобы исключить долгие прогрузы на больших объёмах исторических данных.
- Добавлены таймаут probe и явная пометка медленных ответов в healthcheck (вместо долгого «Проверка...» без статуса).
- В раздел `Совместимость` добавлен агрегированный индикатор деградаций runtime API probes (без ложной кнопки `Исправить` для диагностической записи).
- Обновлён контракт совместимости до `2026-04-08-healthcheck-v2` (UI, backend registry, installer).

### Login / Интерфейс
- Убран лишний pre-check `/api/auth/me` на страницах `login` и `login-2fa`, чтобы не показывать ожидаемую `401 Unauthorized` ошибку до авторизации.

### Startup Self-test / Prod hygiene
- Усилен startup self-test: теперь проверяется полный lifecycle сервиса (`create -> update -> delete`) для `site/upstream/tls/easy profile` с валидацией, что после удаления не остаётся доменных следов в snapshot.
- Для self-test отключены audit side-effects через context, чтобы тест не добавлял записи в `audits`.
- Компиляция для self-test переведена в режим preview (без записи `revisions/jobs/revision-snapshots`), чтобы не засорять persistent storage.
- Добавлена финальная проверка `assertNoStartupSelfTestArtifacts`: после self-test в рабочих storage не должно быть `startup-self-test-*` сущностей, включая тестовые сертификаты.
- Добавлена очистка stale-хвостов `startup-self-test-*` перед запуском self-test (site/upstream/tls/profile/policies/certs).

### Интерфейс / Запросы
- Фильтры `Сервис`, `Метод`, `Статус`, `Время` размещены в первой строке.
- Для выпадающих списков на вкладке `Запросы` добавлены ограничения ширины и фиксированная сетка, чтобы поля не разъезжались по ширине при выборе разных значений.

## [1.1.12] - 2026-04-08

### Запросы / Производительность / Дневные снапшоты
- Переработан runtime-источник `/requests`: вместо полного чтения `access.log` на каждый запрос добавлен дневной архив (`YYYY-MM-DD.jsonl`) с дозаписью новых строк.
- Добавлен фоновый индексатор runtime: архив обновляется автоматически в процессе работы (без открытия вкладки `Запросы`), а при смене даты запись продолжается в новый дневной файл.
- В API запросов добавлена серверная поддержка дневной выборки и ретенции через параметры `day`, `since`, `limit`, `offset`, `retention_days`.
- Вкладка `Запросы` переведена на дневной режим загрузки (по выбранной дате), чтобы не тянуть весь поток событий и снизить нагрузку при открытии страницы.
- Добавлен fallback для read-only runtime storage: если `/var/lib/waf` недоступен для записи, архив запросов автоматически переключается на writable путь в `/tmp`, чтобы `/requests` не падал с `502`.
- Добавлен endpoint runtime `/requests/indexes` для просмотра индексов/снапшотов по дням.

### Настройки / Хранение
- В `Настройки` добавлена новая вкладка `Хранение` рядом с `Основными`.
- Добавлены политики хранения: `Логи запросов`, `Лента активности`, `События`, `Данные банов` (в днях).
- Настройки хранения сохраняются в persistent storage control-plane и применяются в API-выдаче для `requests/events/activity`.
- Вкладка `Хранение` дополнена таблицей индексов/снапшотов (до 10 строк на страницу, далее пагинация).
- Для блока `строки хранения` добавлена отдельная явная кнопка `Сохранить` для применения изменений.
- В таблице индексов добавлено удаление по каждому дню: можно удалить отдельный дневной индекс/снапшот прямо из UI.

### Совместимость / Миграция
- Расширен список legacy-переноса хранилищ: добавлен перенос каталога `settings` в новую структуру storage.
- Обновлён модульный маппинг совместимости для `settings`, чтобы миграция старых данных выполнялась автоматически без ручных фиксов после обновления.

### Сервисы / Удаление
- Исправлена кнопка `Удалить выбранные` в списке сервисов: обработчик перенесен в список (`list view`), массовое удаление снова работает.
- Сценарий удаления сервиса усилен проверками существования связанных сущностей перед `DELETE`, чтобы не отправлять лишние запросы к отсутствующим ресурсам и снизить шум `404` в консоли.
- Для шагов удаления с авто-применением добавлена верификация факта удаления после `400`, чтобы не прерывать очистку, если ресурс уже удалён в control-plane состоянии.

### Баны / Разбан
- Изменено поведение кнопки `Разбанить` в таблице банов: модальное окно больше не открывается, разбан выполняется сразу (с подтверждением).
- Разбан больше не добавляет IP в allowlist/белые списки.
- После разбана запись о бане удаляется из текущего списка в интерфейсе.

### Интерфейс / Навигация / Производительность
- Исправлено «догоняющее» обновление страниц после смены вкладки: добавлена lifecycle-изоляция рендера страницы и отмена предыдущего рендера при навигации.
- Запросы, запущенные на неактивной странице, больше не должны перерисовывать контент после ухода со вкладки.

### Интерфейс / Запросы
- Для колонки `Сервис` добавлена нормализация отображения до host-формата (например, `waf_hantico_ru` -> `waf.hantico.ru`), с приоритетом `primary_host` из `/api/sites`.
- Над поиском добавлены фильтры: `Сервис`, `Метод`, `Статус`, `Время`.
- В фильтре `Время` добавлены режимы: `за минуту`, `за день`, `за месяц`, `определенная дата`, `определенная дата + время`.
- Компоновка фильтров обновлена: `Сервис`, `Метод`, `Статус` отображаются в первой строке, `Время` и выбор даты/даты-времени вынесены во вторую строку.
- В терминологии upstream-полей UI/валидации заменено «хост» на более точное «Адрес upstream» (включая i18n RU/EN).

### Интерфейс / Dashboard / Виджеты
- В меню `Виджеты` добавлена закреплённая кнопка `Сохранить` под списком, чтобы применять изменения явно.
- Сохранение видимости виджетов и лейаута переведено в `localStorage`, поэтому отключенные виджеты корректно сохраняются после перезагрузки страницы.

### Версия
- Версия приложения обновлена до `1.1.12` во всех пользовательских точках отображения и в metadata backend.

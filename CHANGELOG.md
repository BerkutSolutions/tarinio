## [1.5.6] - 20.07.2026

### Runtime fixes

- Runtime waits for the active revision files to finish publishing when the pointer appears before the bundle is complete. This removes an E2E and DAST startup race.

### Сервисы и конфигурация

- Исправлена синхронизация простого и сырого редакторов сервиса. Любое изменение в простом режиме сразу отражается в `WAF_SITE_*`, а изменение в сыром режиме возвращается в простой draft и сохраняется в ресурсах сервиса.
- Изменение upstream, например `127.0.0.1:8080` на `privatebin:8080`, теперь проходит через API, скомпилированную ревизию и активную конфигурацию runtime без сохранения старого значения.
- Белые и чёрные списки больше не перезаписываются compatibility-bridge профиля Easy Site. Основная policy `<site>-access` имеет приоритет над технической `easy-<site>-access`.
- При сохранении из обоих вариантов редактора сначала сохраняется профиль Easy Site, затем каноническая access policy, после чего выполняется единый compile/apply.

### Защита и runtime

- Runtime ожидает фактической готовности HTTP-прокси на активной ревизии после перезагрузки Nginx. Это устраняет переходное окно между старыми и новыми worker-процессами.
- Launcher сохраняет защищённые health и reload endpoint при аварийном завершении контролируемого Nginx, что позволяет откатить применение на последнюю рабочую ревизию.
- Улучшена изоляция Basic Auth между сайтами, обработка graceful reload и проверка разрешённых HTTP-методов для endpoint проверки входа.
- Усилены проверки trusted proxy: подмена `X-Forwarded-For` контролируется для IPv4, IPv6, доверенных и недоверенных цепочек.
- E2E подтверждает реальные режимы защиты AntiBot, ModSecurity, rate limit, L4/L7 adaptive protection, rollback и отказоустойчивость runtime.

### Интерфейс и наблюдаемость

- Исправлено раскрытие сохранённого пароля Basic Auth: поле маскируется по фактической длине, а показ значения выполняется отдельным авторизованным и аудитируемым запросом.
- В «Запросах» добавлен фильтр причины блокировки. Блокировки Basic Auth и AntiBot получают причины `auth` и `antibot`; служебные legacy-поля исключены из интерфейса.
- Дашборд использует единый агрегат защищаемого трафика за 24 часа; средняя отметка шкалы равна половине пика.
- Добавлены девять локализованных вариантов страницы Basic Auth, улучшены предпросмотры, логотипы, контраст и выравнивание селекторов AntiBot и аутентификации.
- Кастомные страницы ошибок получают request ID, IP клиента и время через `Server-Timing`.

### E2E и доказательства

- Добавлен E2E-контракт реального защищаемого сервиса: ресурсы UI/API, candidate revision, active runtime и фактический HTTP-результат должны совпадать. Разрешённый IP получает `200`, неразрешённый — `403`.
- Добавлены контракты полного round-trip для всех полей simple draft и переключения `Простой → Сырой → Простой`; старое значение upstream не может остаться в одном из представлений.
- Каждый E2E job формирует очищенные JSON и Markdown доказательства с тестами, HTTP-статусами, revision ID, checksum конфигурации и причинами блокировок.
- При падении E2E сохраняются состояние Compose и диагностические логи runtime, control-plane и sentinel до teardown.
- Smoke, security-invariants, full и nightly разделены по назначению; пропущенные E2E не считаются доказательством исправности.

### DAST и безопасность зависимостей

- DAST запускает OWASP ZAP только против одноразового E2E-контура WAF и сохраняет JSON, HTML и Markdown отчёты. High и Critical находки блокируют релиз.
- Исправлены запуск ZAP с writable home, обработка Automation Framework и разбор числовых risk code в доказательстве DAST.
- Добавлены негативные security-пробы SQLi, XSS, path traversal, административных API без сессии и легитимных запросов.
- CI выполняет Gitleaks, Trivy, govulncheck и npm audit; обновлена transitive-зависимость `websocket-driver` для устранения CVE-2026-54466.

### Публикация и CI/CD

- После успешных проверок доступны ручные jobs зеркалирования кода в GitHub и публикации подписанного GitHub Release с Docker-образом в GHCR.
- GitHub Release теперь содержит единый `release-evidence.md/json` и архив исходных очищенных E2E/DAST-доказательств. Публикация завершается ошибкой, если E2E содержат fail/skip или DAST сообщает High/Critical.
- Перед публикацией образа проверяются сгенерированные Nginx-артефакты, подпись, SBOM, provenance и `/healthz` собранного Docker-образа.

## [2.0.4] - 22.04.2026

### Инициализация и доступ
- Исправлен first-run onboarding: первичная настройка больше не запускает преждевременный auto-apply на промежуточных шагах.
- Переход на `https://` выполняется только после успешной компиляции и применения ревизии с подтверждённым `has_active_revision=true`.
- Если первичная инициализация не завершена, страницы входа и приложение возвращают пользователя в `http` onboarding вместо сломанного HTTPS-логина.
- Снаружи больше не торчит отдельный UI-порт для администрирования после первичной настройки; bootstrap-доступ идёт через runtime gateway.

### Интерфейс и сервисы
- Исправлены виджеты дашборда после перехода на PostgreSQL: статистика атак и fallback по blocked-запросам снова наполняют карточки и детали атак.
- Детализация загрузки CPU и памяти в дашборде переведена с “топа сайтов” на реальные процессы системы.
- В разделе `Сервисы` добавлен сырой режим редактирования `Raw`, который работает как редактирование полного `.env`-описания сервиса.
- Raw-режим автоматически подставляет текущее состояние сервиса при переключении из easy-редактора и сохраняет изменения через ту же валидацию и тот же pipeline ревизий.
- Экспорт `.env` для сервисов усилен: теперь он гидратирует полный draft сервиса и сохраняет не только базовые поля, но и параметры easy-profile, access policy и route-based лимиты, включая `WAF_SITE_CUSTOM_LIMIT_RULES`.

### Диагностика и администрирование
- В control-plane добавлено расширенное структурированное логирование HTTP-запросов без секретов: метод, маршрут, путь, код ответа, длительность, IP, forwarded-for, request id, user agent и ключи query.
- `Сборщик событий WAF` переведён на безаутентификационный запуск по аналогии со `Сборщиком ошибок`; поля логина и пароля удалены из каталога скриптов и UI.
- `Сборщик событий WAF` и `Сборщик ошибок` дополнительно собирают более полезные срезы по URI, status codes, recent access, ingest-паттернам и runtime/control-plane request traces.
- Для административных скриптов сохранена поддержка диапазона `SINCE`, фильтрации по URI и более подробных diagnostic artifacts для расследований.

- Версия продукта, интерфейса, документации и release metadata синхронизирована на `2.0.4`.
- Полностью обновлён changelog релиза и приведён к структуре по тематическим главам.
